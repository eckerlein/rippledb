# Installation

RippleDB is distributed as a set of npm packages. Install what you need based on your use case.

## Server-Side Setup

For a backend that stores changes and materializes state:

<Tabs items={['pnpm', 'npm', 'yarn']}>
  <Tab value="pnpm">
```bash
pnpm add @rippledb/core @rippledb/server @rippledb/db-sqlite @rippledb/materialize-db
```
  </Tab>
  <Tab value="npm">
```bash
npm install @rippledb/core @rippledb/server @rippledb/db-sqlite @rippledb/materialize-db
```
  </Tab>
  <Tab value="yarn">
```bash
yarn add @rippledb/core @rippledb/server @rippledb/db-sqlite @rippledb/materialize-db
```
  </Tab>
</Tabs>

### Using Drizzle ORM?

If you're already using Drizzle, use the Drizzle-specific packages instead:

<Tabs items={['pnpm', 'npm', 'yarn']}>
  <Tab value="pnpm">
```bash
pnpm add @rippledb/core @rippledb/server @rippledb/db-drizzle @rippledb/materialize-drizzle
```
  </Tab>
  <Tab value="npm">
```bash
npm install @rippledb/core @rippledb/server @rippledb/db-drizzle @rippledb/materialize-drizzle
```
  </Tab>
  <Tab value="yarn">
```bash
yarn add @rippledb/core @rippledb/server @rippledb/db-drizzle @rippledb/materialize-drizzle
```
  </Tab>
</Tabs>

## Client-Side Setup

For a client that syncs with a RippleDB server:

<Tabs items={['pnpm', 'npm', 'yarn']}>
  <Tab value="pnpm">
```bash
pnpm add @rippledb/core @rippledb/client @rippledb/remote-http
```
  </Tab>
  <Tab value="npm">
```bash
npm install @rippledb/core @rippledb/client @rippledb/remote-http
```
  </Tab>
  <Tab value="yarn">
```bash
yarn add @rippledb/core @rippledb/client @rippledb/remote-http
```
  </Tab>
</Tabs>

## Package Overview

| Package | Purpose | Docs |
|---------|---------|------|
| `@rippledb/core` | HLC timestamps, Change types, merge logic (always required) | [/docs/reference/core](/docs/reference/core) |
| `@rippledb/server` | Db interface and contracts for server persistence | [/docs/reference/server](/docs/reference/server) |
| `@rippledb/client` | Client-side sync orchestration and Store contract | [/docs/reference/client](/docs/reference/client) |
| `@rippledb/remote-http` | HTTP Remote implementation (client ↔ server sync) | [/docs/reference/remote-http](/docs/reference/remote-http) |
| `@rippledb/remote-trpc` | tRPC Remote implementation (client ↔ server sync) | [/docs/reference/remote-trpc](/docs/reference/remote-trpc) |
| `@rippledb/server-trpc` | tRPC router exposing `pull`/`append` procedures | [/docs/reference/server-trpc](/docs/reference/server-trpc) |
| `@rippledb/zod` | Zod schemas for runtime validation of RippleDB payloads | [/docs/reference/zod](/docs/reference/zod) |
| `@rippledb/db-sqlite` | SQLite adapter using better-sqlite3 | [/docs/adapters/db-sqlite](/docs/adapters/db-sqlite) |
| `@rippledb/db-turso` | Turso/libSQL adapter | [/docs/adapters/db-turso](/docs/adapters/db-turso) |
| `@rippledb/db-drizzle` | Database-agnostic Drizzle ORM adapter | [/docs/adapters/db-drizzle](/docs/adapters/db-drizzle) |
| `@rippledb/db-memory` | In-memory Db adapter (tests) | [/docs/adapters/db-memory](/docs/adapters/db-memory) |
| `@rippledb/materialize-core` | Core materialization algorithm + adapter interfaces | [/docs/adapters/materialize-core](/docs/adapters/materialize-core) |
| `@rippledb/materialize-db` | SQL-based state materializer (raw SQL) | [/docs/adapters/materialize-db](/docs/adapters/materialize-db) |
| `@rippledb/materialize-drizzle` | Drizzle-based state materializer (type-safe) | [/docs/adapters/materialize-drizzle](/docs/adapters/materialize-drizzle) |
| `@rippledb/store-memory` | In-memory Store implementation (tests) | [/docs/adapters/store-memory](/docs/adapters/store-memory) |
| `@rippledb/bind-tanstack` | TanStack Query cache invalidation binding | [/docs/adapters/bind-tanstack](/docs/adapters/bind-tanstack) |

## TypeScript

RippleDB is written in TypeScript and ships with full type definitions. No additional `@types/*` packages needed.
