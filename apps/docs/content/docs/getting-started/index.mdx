# Installation

RippleDB is distributed as a set of npm packages. Install what you need based on your use case.

## Server-Side Setup

For a backend that stores changes and materializes state:

<Tabs items={['pnpm', 'npm', 'yarn']}>
  <Tab value="pnpm">
```bash
pnpm add @rippledb/core @rippledb/server @rippledb/db-sqlite @rippledb/materialize-db
```
  </Tab>
  <Tab value="npm">
```bash
npm install @rippledb/core @rippledb/server @rippledb/db-sqlite @rippledb/materialize-db
```
  </Tab>
  <Tab value="yarn">
```bash
yarn add @rippledb/core @rippledb/server @rippledb/db-sqlite @rippledb/materialize-db
```
  </Tab>
</Tabs>

## Package Overview

| Package | Purpose |
|---------|---------|
| `@rippledb/core` | HLC timestamps, Change types, merge logic (always required) |
| `@rippledb/server` | Db interface and contracts for server persistence |
| `@rippledb/client` | Client-side sync orchestration |
| `@rippledb/db-sqlite` | SQLite adapter using better-sqlite3 |
| `@rippledb/db-turso` | Turso/libSQL adapter |
| `@rippledb/db-drizzle` | Database-agnostic Drizzle ORM adapter |
| `@rippledb/materialize-db` | SQL-based state materializer |
| `@rippledb/materialize-drizzle` | Drizzle-based state materializer |
| `@rippledb/remote-http` | HTTP client for sync |

## TypeScript

RippleDB is written in TypeScript and ships with full type definitions. No additional `@types/*` packages needed.
